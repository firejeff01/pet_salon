## 通用規範

### 成功回應格式

直接回傳資料 JSON（物件或陣列），不包裝在額外欄位中。

```json
// 單筆資料
{ "id": "owner_001", "name": "王小明", ... }

// 列表資料
[{ "id": "owner_001", ... }, { "id": "owner_002", ... }]
```

### 錯誤回應格式

```json
{
  "error": {
    "code": 400,
    "message": "姓名為必填欄位"
  }
}
```

### HTTP Status Code 定義

| Status Code | 說明 |
|---|---|
| 200 | 成功（查詢、更新） |
| 201 | 成功建立資源 |
| 400 | 請求參數錯誤、驗證失敗 |
| 404 | 資源不存在 |
| 500 | 伺服器內部錯誤（含 PDF 產生失敗、檔案權限問題） |

### 時間格式

所有時間欄位使用 ISO 8601 格式：`2026-04-16T12:00:00+08:00`

## 1. 飼主 API

### 1.1 GET /api/owners

取得飼主列表，支援關鍵字搜尋。

**Query Parameters：**

| 參數 | 型別 | 必填 | 說明 |
|---|---|---|---|
| keyword | string | 否 | 搜尋關鍵字，依姓名或電話模糊比對 |

**Response 200：**

```json
[
  {
    "id": "owner_001",
    "name": "王小明",
    "nationalId": "A123456789",
    "phone": "0912345678",
    "address": "桃園市桃園區中正路1號",
    "emergencyContactName": "王太太",
    "emergencyContactPhone": "0987654321",
    "emergencyContactRelationship": "配偶",
    "preferredAnimalHospital": "仁愛動物醫院",
    "preferredAnimalHospitalPhone": "03-1234567",
    "preferredAnimalHospitalAddress": "桃園市中壢區中正路100號",
    "isStoredValueCustomer": true,
    "storedValueBalance": 5000,
    "note": "",
    "createdAt": "2026-04-16T12:00:00+08:00",
    "updatedAt": "2026-04-16T12:00:00+08:00"
  }
]
```

### 1.2 GET /api/owners/:ownerId

取得單一飼主完整資料。

**Path Parameters：**

| 參數 | 型別 | 說明 |
|---|---|---|
| ownerId | string | 飼主 ID |

**Response 200：** 同上單筆

**Response 404：**

```json
{ "error": { "code": 404, "message": "查無此飼主" } }
```

### 1.3 POST /api/owners

新增飼主。

**Request Body：**

```json
{
  "name": "王小明",
  "nationalId": "A123456789",
  "phone": "0912345678",
  "address": "桃園市桃園區中正路1號",
  "emergencyContactName": "王太太",
  "emergencyContactPhone": "0987654321",
  "emergencyContactRelationship": "配偶",
  "preferredAnimalHospital": "",
  "preferredAnimalHospitalPhone": "",
  "preferredAnimalHospitalAddress": "",
  "isStoredValueCustomer": false,
  "storedValueBalance": 0,
  "note": ""
}
```

**驗證規則：**

| 欄位 | 規則 |
|---|---|
| name | 必填，非空字串 |
| nationalId | 必填，非空字串 |
| phone | 必填，非空字串 |
| address | 必填，非空字串 |
| emergencyContactName | 必填，非空字串 |
| emergencyContactPhone | 必填，非空字串 |
| emergencyContactRelationship | 必填，非空字串 |
| storedValueBalance | 不得小於 0 |

**Response 201：** 回傳含自動產生 id、createdAt、updatedAt 的完整飼主資料

**Response 400：**

```json
{ "error": { "code": 400, "message": "name 為必填欄位" } }
```

### 1.4 PUT /api/owners/:ownerId

更新飼主資料（含儲值金額）。

**Request Body：** 同 POST，但為部分或全部欄位更新

**Response 200：** 回傳更新後的完整飼主資料（updatedAt 已更新）

**Response 400：** 驗證錯誤

**Response 404：** 查無此飼主

## 3. 預約 API

### 3.1 GET /api/appointments

取得預約列表，支援日期區間查詢。

**Query Parameters：**

| 參數 | 型別 | 必填 | 說明 |
|---|---|---|---|
| startDate | string (YYYY-MM-DD) | 否 | 查詢起始日期 |
| endDate | string (YYYY-MM-DD) | 否 | 查詢結束日期 |

**Response 200：**

```json
[
  {
    "id": "appt_001",
    "ownerId": "owner_001",
    "petId": "pet_001",
    "date": "2026-04-20",
    "time": "14:00",
    "status": "已預約",
    "cancelReason": "",
    "note": "",
    "createdAt": "2026-04-16T12:00:00+08:00",
    "updatedAt": "2026-04-16T12:00:00+08:00"
  }
]
```

### 3.2 GET /api/appointments/date/:date

取得指定日期的所有預約（含關聯的飼主姓名、電話、寵物名稱、是否已產生契約），依時間升序排列。

**Response 200：**

```json
[
  {
    "id": "appt_001",
    "ownerId": "owner_001",
    "petId": "pet_001",
    "date": "2026-04-20",
    "time": "10:00",
    "status": "已預約",
    "cancelReason": "",
    "note": "",
    "petName": "Mimi",
    "ownerName": "李小花",
    "ownerPhone": "0911222333",
    "hasContract": false,
    "createdAt": "...",
    "updatedAt": "..."
  }
]
```

### 3.3 POST /api/appointments

新增預約。

**Request Body：**

```json
{
  "ownerId": "owner_001",
  "petId": "pet_001",
  "date": "2026-04-20",
  "time": "14:00",
  "note": ""
}
```

**驗證規則：**

| 欄位 | 規則 |
|---|---|
| ownerId | 必填 |
| petId | 必填 |
| date | 必填，YYYY-MM-DD 格式 |
| time | 必填，HH:mm 格式 |

**Response 201：** status 自動設為 "已預約"

### 3.4 PUT /api/appointments/:appointmentId

更新預約（含取消與取消原因）。

**Request Body：**

```json
{
  "date": "2026-04-21",
  "time": "10:00",
  "status": "已取消",
  "cancelReason": "飼主臨時有事",
  "note": ""
}
```

**驗證規則：**

| 規則 | 說明 |
|---|---|
| status 為 "已取消" 時 cancelReason 必填 | 取消原因不得為空 |

    "limbs": "正常",
    "skin": "正常",
    "fur": "正常"
  },
  "medicalHistory": ["心臟病"],
  "medicalHistoryOther": "",
  "personality": ["親人", "不會咬人"],
  "ownerNotes": "",
  "shopNotes": "",

### 4.2 POST /api/grooming-records

建立美容紀錄。

**Request Body：**

```json
{
  "appointmentId": "appt_001",
  "serviceDate": "2026-04-20",
  "serviceTime": "14:00",
  "services": [
    { "item": "洗澡", "price": 500 }
  ],
  "physicalExamination": {
    "eyes": "正常",
    "ears": "正常",
    "teeth": "正常",
    "limbs": "正常",
    "skin": "正常",
    "fur": "正常"
  },
  "medicalHistory": [],
  "medicalHistoryOther": "",
  "personality": ["親人", "不會咬人"],
  "ownerNotes": "",
  "shopNotes": "",
  "otherNotes": ""
}
```

**驗證規則：**

| 欄位 | 規則 |
|---|---|
| appointmentId | 必填，對應預約必須存在 |
| services | 必填，非空陣列 |
| services[].item | 必填，值必須為「洗澡」「美容」「其他」之一 |
| services[].price | 必填，數值 >= 0 |
| physicalExamination | 必填 |
| personality | 必填，非空陣列 |

**後端自動計算：**

- storedValueDeduction：儲值型客戶自動計算
- cashPayment：自動計算（總費用 - 儲值扣款）
- 儲值型客戶的 storedValueBalance 自動扣除

**Response 201：** 含 id、storedValueDeduction、cashPayment

### 4.3 PUT /api/grooming-records/:recordId

更新美容紀錄。

## 5. 契約 API

### 5.1 POST /api/contracts/generate

產生契約 PDF。

**Request Body：**

```json
{
  "recordId": "rec_001"
}
```

**Response 201：**

```json
{
  "contractPath": "generated-contracts/20260420_1400_Lucky_王小明_契約.pdf",
  "version": 1,
  "generatedAt": "2026-04-20T14:30:00+08:00",
  "generatedBy": "美容師"
}
```

**錯誤：**

| Status | 情境 | error.message |
|---|---|---|
| 400 | 必填欄位不完整 | "必填欄位不完整：..." |
| 404 | 美容紀錄不存在 | "查無美容紀錄" |
| 500 | PDF 模板不存在 | "PDF 模板不存在，請確認模板路徑設定" |
| 500 | 輸出目錄無寫入權限 | "無法寫入輸出目錄" |
| 500 | PDF 產生失敗 | "契約 PDF 產生失敗" |

### 5.2 GET /api/contracts/:recordId

下載最新版本的契約 PDF。

**Response 200：** Content-Type: application/pdf，回傳 PDF 檔案串流

**Response 404：** 契約 PDF 不存在

### 5.3 GET /api/contracts/:recordId/versions

取得美容紀錄的所有契約版本。

**Response 200：**

```json
[
  {
    "version": 1,
    "path": "generated-contracts/20260420_1400_Lucky_王小明_契約.pdf",
    "generatedAt": "2026-04-20T14:30:00+08:00",
    "generatedBy": "美容師"
  },
  {
    "version": 2,
    "path": "generated-contracts/20260420_1400_Lucky_王小明_契約_v2.pdf",
    "generatedAt": "2026-04-20T15:00:00+08:00",
    "generatedBy": "美容師"
  }
]
```

### 5.4 POST /api/contracts/:recordId/sign

上傳電子簽名並嵌入 PDF。

**Request Body：**

```json
{
  "signatureImage": "data:image/png;base64,iVBORw0KGgo..."
}
```

**驗證規則：**

| 欄位 | 規則 |
|---|---|
| signatureImage | 必填，非空字串，base64 PNG 格式 |

**Response 200：**

```json
{
  "contractPath": "generated-contracts/20260420_1400_Lucky_王小明_契約.pdf",
  "message": "簽名已成功嵌入 PDF"
}
```

**錯誤：**

| Status | 情境 | error.message |
|---|---|---|
| 400 | 簽名圖片為空 | "簽名圖片為必填" |
| 404 | 契約 PDF 不存在 | "契約 PDF 不存在，請先產生契約" |

## 6. 備份 API

### 6.1 POST /api/backup

執行一鍵備份。

**Response 200：**

```json
{
  "backupPath": "backups/backup_20260416_120000.zip",
  "timestamp": "2026-04-16T12:00:00+08:00",
  "filesCount": 8
}
```

**錯誤：**

| Status | 情境 | error.message |
|---|---|---|
| 500 | 無寫入權限 | "無法寫入備份目錄，請檢查目錄權限" |

### 6.2 GET /api/backup/list

取得備份檔案列表。

**Response 200：**

```json
[
  {
    "fileName": "backup_20260416_120000.zip",
    "timestamp": "2026-04-16T12:00:00+08:00",
    "size": 1048576
  }
]
```
