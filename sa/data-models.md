## 1. Owner（飼主）

**檔案：** `data/owners.json`

**結構：**

```json
{
  "owners": [
    {
      "id": "owner_001",
      "name": "王小明",
      "nationalId": "A123456789",
      "phone": "0912345678",
      "address": "桃園市桃園區中正路1號",
      "emergencyContactName": "王太太",
      "emergencyContactPhone": "0987654321",
      "emergencyContactRelationship": "配偶",
      "preferredAnimalHospital": "仁愛動物醫院",
      "preferredAnimalHospitalPhone": "03-1234567",
      "preferredAnimalHospitalAddress": "桃園市中壢區中正路100號",
      "isStoredValueCustomer": true,
      "storedValueBalance": 5000,
      "note": "VIP客戶",
      "createdAt": "2026-04-16T12:00:00+08:00",
      "updatedAt": "2026-04-16T12:00:00+08:00"
```

**欄位定義：**

| 欄位 | 型別 | 必填 | 預設值 | 說明 |
|---|---|---|---|---|
| id | string | 是（系統產生） | - | 格式 "owner_" + 唯一識別碼 |
| name | string | 是 | - | 飼主姓名 |
| nationalId | string | 是 | - | 身分證字號 |
| phone | string | 是 | - | 聯絡電話 |
| address | string | 是 | - | 通訊地址 |
| emergencyContactName | string | 是 | - | 緊急聯絡人姓名 |
| emergencyContactPhone | string | 是 | - | 緊急聯絡人電話 |
| emergencyContactRelationship | string | 是 | - | 緊急聯絡人與飼主關係 |
| preferredAnimalHospital | string | 否 | "" | 飼主指定獸醫院名稱 |
| preferredAnimalHospitalPhone | string | 否 | "" | 飼主指定獸醫院電話 |
| preferredAnimalHospitalAddress | string | 否 | "" | 飼主指定獸醫院地址 |
| isStoredValueCustomer | boolean | 是 | false | 是否為儲值型客戶 |
| storedValueBalance | number | 否 | 0 | 儲值餘額，不得小於 0 |
| note | string | 否 | "" | 備註 |
| createdAt | string (ISO 8601) | 是（系統產生） | - | 建立時間 |
| updatedAt | string (ISO 8601) | 是（系統產生） | - | 更新時間 |

**關聯：**

- 一位飼主可有多隻寵物（1:N，透過 Pet.ownerId 關聯）
- 一位飼主可有多筆預約（1:N，透過 Appointment.ownerId 關聯）

## 2. Pet（寵物）

**檔案：** `data/pets.json`

**結構：**

```json
{
  "pets": [
    {
      "id": "pet_001",
      "ownerId": "owner_001",
      "name": "Lucky",
      "species": "犬",
      "breed": "貴賓",
      "gender": "公",
      "age": "3歲",
      "isNeutered": true,
      "chipNumber": "900000000012345",
      "unregisteredIdMethod": "",
      "personality": ["親人", "不會咬人", "不會咬狗貓"],
      "physicalExamination": {
        "eyes": "正常",
        "ears": "正常",
        "teeth": "正常",
        "limbs": "正常",
        "skin": "正常",
        "fur": "正常"
      },
      "medicalHistory": ["心臟病"],
      "medicalHistoryOther": "",
      "note": "",
      "createdAt": "2026-04-16T12:00:00+08:00",
      "updatedAt": "2026-04-16T12:00:00+08:00"
    }
  ]

```

**欄位定義：**

| 欄位 | 型別 | 必填 | 預設值 | 說明 |
|---|---|---|---|---|
| id | string | 是（系統產生） | - | 格式 "pet_" + 唯一識別碼 |
| ownerId | string | 是 | - | 關聯飼主 ID |
| name | string | 是 | - | 寵物名稱 |
| species | string | 是 | - | 種類，值為「犬」或「貓」 |
| breed | string | 是 | - | 品種 |
| gender | string | 是 | - | 性別，值為「公」或「母」 |
| age | string | 是 | - | 年齡（如 "3歲"） |
| isNeutered | boolean | 是 | - | 是否結育 |
| chipNumber | string | 否 | "" | 晶片號碼 |
| unregisteredIdMethod | string | 否 | "" | 未登記身分識別方式 |
| personality | string[] | 是 | - | 寵物個性（多選） |
| physicalExamination | object | 是 | - | 身體狀態檢視 |
| physicalExamination.eyes | string | 是 | "正常" | 眼睛，值為「正常」/「異常」 |
| physicalExamination.ears | string | 是 | "正常" | 耳朵，值為「正常」/「異常」 |
| physicalExamination.teeth | string | 是 | "正常" | 牙齒，值為「正常」/「異常」 |
| physicalExamination.limbs | string | 是 | "正常" | 四肢，值為「正常」/「異常」 |
| physicalExamination.skin | string | 是 | "正常" | 皮膚，值為「正常」/「異常」 |
| physicalExamination.fur | string | 是 | "正常" | 皮毛，值為「正常」/「打結」/「跳蚤壁蝨」 |
| medicalHistory | string[] | 是 | [] | 寵物病史（多選，可為空陣列） |
| medicalHistoryOther | string | 否 | "" | 病史「其它」的自由輸入內容 |
| note | string | 否 | "" | 備註 |
| createdAt | string (ISO 8601) | 是（系統產生） | - | 建立時間 |
| updatedAt | string (ISO 8601) | 是（系統產生） | - | 更新時間 |

**personality 固定選項：**

`["親人", "不會咬人", "不會咬狗貓", "容易緊張", "親狗", "會咬人", "會咬狗貓", "有攻擊性"]`

**medicalHistory 固定選項：**

`["心臟病", "氣喘", "氣管塌陷", "白內障", "癲癇", "心絲蟲", "艾利希體", "腹膜炎", "腹積水", "手術外傷未癒合", "髖關節問題", "骨折", "腸炎", "血便", "血尿", "懷孕", "傳染性疾病", "其它"]`

**關聯：**

- 每隻寵物必須關聯一位飼主（N:1，透過 ownerId）
- 一隻寵物可有多筆預約（1:N，透過 Appointment.petId）

## 3. Appointment（預約）

**檔案：** `data/appointments.json`

**結構：**

```json
{
  "appointments": [
    {
      "id": "appt_001",
      "ownerId": "owner_001",
      "petId": "pet_001",
      "date": "2026-04-20",
      "time": "14:00",
      "status": "已預約",
      "cancelReason": "",
      "note": "客戶要求使用低敏洗毛精",
      "createdAt": "2026-04-16T12:00:00+08:00",
      "updatedAt": "2026-04-16T12:00:00+08:00"
    }
  ]
}
```

**欄位定義：**

| 欄位 | 型別 | 必填 | 預設值 | 說明 |
|---|---|---|---|---|
| id | string | 是（系統產生） | - | 格式 "appt_" + 唯一識別碼 |
| ownerId | string | 是 | - | 關聯飼主 ID |
| petId | string | 是 | - | 關聯寵物 ID |
| date | string (YYYY-MM-DD) | 是 | - | 預約日期 |
| time | string (HH:mm) | 是 | - | 預約時間 |
| status | string | 是 | "已預約" | 預約狀態，值為「已預約」/「已完成」/「已取消」 |
| cancelReason | string | 取消時必填 | "" | 取消原因 |
| note | string | 否 | "" | 預約備註 |
| createdAt | string (ISO 8601) | 是（系統產生） | - | 建立時間 |
| updatedAt | string (ISO 8601) | 是（系統產生） | - | 更新時間 |

**關聯：**

- 每筆預約關聯一位飼主（N:1，透過 ownerId）
- 每筆預約關聯一隻寵物（N:1，透過 petId）
- 每筆預約可有一筆美容紀錄（1:1，透過 GroomingRecord.appointmentId）

## 資料關聯圖

```
Owner (1) ──── (N) Pet
  │                  │
  │                  │
  (N)               (N)
  │                  │
Appointment (1) ─── (1) GroomingRecord
                          │
                          (N)
                          │
                     ContractPDF (版本)
```

- 一位飼主可有多隻寵物
- 一位飼主可有多筆預約
- 一隻寵物可有多筆預約
- 同一飼主的多隻寵物可在同日有多筆預約
- 每筆預約對應最多一筆美容紀錄
- 每筆美容紀錄可有多個版本的契約 PDF

## ID 產生規則

| 資料類型 | ID 前綴 | 範例 |
|---|---|---|
| 飼主 | owner_ | owner_001, owner_abc123 |
| 寵物 | pet_ | pet_001, pet_abc123 |
| 預約 | appt_ | appt_001, appt_abc123 |
| 美容紀錄 | rec_ | rec_001, rec_abc123 |

建議使用 UUID v4 或 nanoid 產生唯一識別碼，前綴後接產生的識別碼。

## 契約 PDF 檔名規則

**格式：** `YYYYMMDD_HHmm_寵物名稱_飼主姓名_契約.pdf`

**版本命名：** `YYYYMMDD_HHmm_寵物名稱_飼主姓名_契約_v{N}.pdf`

**範例：**
- 首次產生：`20260420_1400_Lucky_王小明_契約.pdf`
- 第二次產生：`20260420_1400_Lucky_王小明_契約_v2.pdf`
- 第三次產生：`20260420_1400_Lucky_王小明_契約_v3.pdf`

**特殊字元處理：** 檔名中不得包含 `\ / : * ? " < > |`，遇到時替換為底線 `_`。
